The Airbyte API lets you interact with Airbyte programmatically. Use it to automate data pipeline management, integrate with orchestration tools like Airflow, or build on top of [Airbyte Embedded](https://airbyte.com/ai).

## Get an access token

Before making requests you need an access token. See [configuring API access](/developers/api-documentation) for setup instructions.

## Base URL

The base URL depends on your deployment:

| Deployment | Base URL |
|---|---|
| Airbyte Cloud | `https://api.airbyte.com/v1/` |
| Self-managed (localhost) | `http://localhost:8000/api/public/v1/` |
| Self-managed (web) | `<YOUR_AIRBYTE_URL>/api/public/v1/` |

## Core endpoints

### Sources

<AccordionGroup>
  <Accordion title="GET /v1/sources — List sources">
    Returns all source connectors in a workspace.

    **Query parameters**

    <ParamField query="workspaceIds" type="string">
      Comma-separated list of workspace IDs to filter results.
    </ParamField>
    <ParamField query="limit" type="integer">
      Maximum number of results to return. Defaults to 20.
    </ParamField>
    <ParamField query="offset" type="integer">
      Number of results to skip for pagination.
    </ParamField>

    **Response fields**

    <ResponseField name="data" type="array">
      Array of source objects.
    </ResponseField>
    <ResponseField name="data[].sourceId" type="string">
      Unique identifier for the source.
    </ResponseField>
    <ResponseField name="data[].name" type="string">
      Display name of the source.
    </ResponseField>
    <ResponseField name="data[].sourceType" type="string">
      Connector type identifier (e.g., `stripe`, `snowflake`).
    </ResponseField>
    <ResponseField name="data[].workspaceId" type="string">
      Workspace the source belongs to.
    </ResponseField>
    <ResponseField name="data[].configuration" type="object">
      Connector-specific configuration. Sensitive fields are masked.
    </ResponseField>
    <ResponseField name="data[].createdAt" type="integer">
      Unix timestamp of when the source was created.
    </ResponseField>
  </Accordion>

  <Accordion title="POST /v1/sources — Create a source">
    Creates a new source connector.

    ```bash
    curl --request POST \
         --url 'https://api.airbyte.com/v1/sources' \
         --header 'accept: application/json' \
         --header 'authorization: Bearer <YOUR_ACCESS_TOKEN>' \
         --header 'content-type: application/json' \
         --data '{
           "name": "My Stripe Source",
           "workspaceId": "<YOUR_WORKSPACE_ID>",
           "definitionId": "e094cb9a-26de-4645-8761-65c0c425d1de",
           "configuration": {
             "account_id": "<YOUR_STRIPE_ACCOUNT_ID>",
             "client_secret": "<YOUR_STRIPE_SECRET_KEY>",
             "start_date": "2023-01-01T00:00:00Z"
           }
         }'
    ```

    **Body parameters**

    <ParamField body="name" type="string" required>
      Display name for the source.
    </ParamField>
    <ParamField body="workspaceId" type="string" required>
      ID of the workspace to create the source in.
    </ParamField>
    <ParamField body="definitionId" type="string" required>
      UUID of the connector definition. Find this in the connector catalog.
    </ParamField>
    <ParamField body="configuration" type="object" required>
      Connector-specific configuration object.
    </ParamField>
  </Accordion>

  <Accordion title="GET /v1/sources/{sourceId} — Get a source">
    Returns a single source by ID.

    ```bash
    curl --request GET \
         --url 'https://api.airbyte.com/v1/sources/<SOURCE_ID>' \
         --header 'accept: application/json' \
         --header 'authorization: Bearer <YOUR_ACCESS_TOKEN>'
    ```
  </Accordion>

  <Accordion title="PATCH /v1/sources/{sourceId} — Update a source">
    Updates an existing source. Only provide fields you want to change.

    ```bash
    curl --request PATCH \
         --url 'https://api.airbyte.com/v1/sources/<SOURCE_ID>' \
         --header 'accept: application/json' \
         --header 'authorization: Bearer <YOUR_ACCESS_TOKEN>' \
         --header 'content-type: application/json' \
         --data '{"name": "Updated source name"}'
    ```
  </Accordion>

  <Accordion title="DELETE /v1/sources/{sourceId} — Delete a source">
    Permanently deletes a source and all associated connections.

    <Warning>Deleting a source also deletes all connections that use it. This action cannot be undone.</Warning>

    ```bash
    curl --request DELETE \
         --url 'https://api.airbyte.com/v1/sources/<SOURCE_ID>' \
         --header 'authorization: Bearer <YOUR_ACCESS_TOKEN>'
    ```
  </Accordion>
</AccordionGroup>

### Connections

<AccordionGroup>
  <Accordion title="GET /v1/connections — List connections">
    Returns all connections in a workspace.

    ```bash
    curl --request GET \
         --url 'https://api.airbyte.com/v1/connections?workspaceIds=<YOUR_WORKSPACE_ID>' \
         --header 'accept: application/json' \
         --header 'authorization: Bearer <YOUR_ACCESS_TOKEN>'
    ```

    **Response fields**

    <ResponseField name="data[].connectionId" type="string">
      Unique identifier for the connection.
    </ResponseField>
    <ResponseField name="data[].name" type="string">
      Display name of the connection.
    </ResponseField>
    <ResponseField name="data[].sourceId" type="string">
      ID of the source connector.
    </ResponseField>
    <ResponseField name="data[].destinationId" type="string">
      ID of the destination connector.
    </ResponseField>
    <ResponseField name="data[].status" type="string">
      Connection status: `active`, `inactive`, or `deprecated`.
    </ResponseField>
    <ResponseField name="data[].schedule" type="object">
      Sync schedule configuration.
    </ResponseField>
  </Accordion>

  <Accordion title="POST /v1/connections — Create a connection">
    Creates a connection between a source and a destination.

    ```bash
    curl --request POST \
         --url 'https://api.airbyte.com/v1/connections' \
         --header 'accept: application/json' \
         --header 'authorization: Bearer <YOUR_ACCESS_TOKEN>' \
         --header 'content-type: application/json' \
         --data '{
           "name": "Stripe to BigQuery",
           "sourceId": "<YOUR_SOURCE_ID>",
           "destinationId": "<YOUR_DESTINATION_ID>",
           "schedule": {
             "scheduleType": "cron",
             "cronExpression": "0 0 * * *"
           },
           "namespaceDefinition": "destination",
           "prefix": "stripe_"
         }'
    ```
  </Accordion>
</AccordionGroup>

### Jobs

<AccordionGroup>
  <Accordion title="POST /v1/jobs — Trigger a sync">
    Triggers a manual sync job for a connection.

    ```bash
    curl --request POST \
         --url 'https://api.airbyte.com/v1/jobs' \
         --header 'accept: application/json' \
         --header 'authorization: Bearer <YOUR_ACCESS_TOKEN>' \
         --header 'content-type: application/json' \
         --data '{
           "connectionId": "<YOUR_CONNECTION_ID>",
           "jobType": "sync"
         }'
    ```

    ```json
    {
      "jobId": 18",
      "status": "pending",
      "jobType": "sync",
      "startTime": "2024-01-15T10:00:00Z",
      "connectionId": "<YOUR_CONNECTION_ID>"
    }
    ```

    <ParamField body="connectionId" type="string" required>
      ID of the connection to sync.
    </ParamField>
    <ParamField body="jobType" type="string" required>
      Type of job to run. Use `sync` for data replication or `reset` to clear destination data.
    </ParamField>
  </Accordion>

  <Accordion title="GET /v1/jobs/{jobId} — Get job status">
    Returns the current status of a job.

    ```bash
    curl --request GET \
         --url 'https://api.airbyte.com/v1/jobs/<JOB_ID>' \
         --header 'accept: application/json' \
         --header 'authorization: Bearer <YOUR_ACCESS_TOKEN>'
    ```

    ```json
    {
      "jobId": "18",
      "status": "succeeded",
      "jobType": "sync",
      "startTime": "2024-01-15T10:00:00Z",
      "lastUpdatedAt": "2024-01-15T10:04:37Z",
      "connectionId": "<YOUR_CONNECTION_ID>",
      "bytesSynced": 1048576,
      "rowsSynced": 15420
    }
    ```

    <ResponseField name="jobId" type="string">
      Unique identifier for the job.
    </ResponseField>
    <ResponseField name="status" type="string">
      Current job status: `pending`, `running`, `succeeded`, `failed`, or `cancelled`.
    </ResponseField>
    <ResponseField name="bytesSynced" type="integer">
      Total bytes synced in this job run.
    </ResponseField>
    <ResponseField name="rowsSynced" type="integer">
      Total rows synced in this job run.
    </ResponseField>
  </Accordion>

  <Accordion title="DELETE /v1/jobs/{jobId} — Cancel a job">
    Cancels a running job.

    ```bash
    curl --request DELETE \
         --url 'https://api.airbyte.com/v1/jobs/<JOB_ID>' \
         --header 'authorization: Bearer <YOUR_ACCESS_TOKEN>'
    ```
  </Accordion>
</AccordionGroup>

## Handle token expiry

Access tokens are short-lived. To avoid authorization errors in your integration:

- Cache the token along with its expiry time when you first receive it.
- Before each request, check whether the token has expired. If it has, fetch a new one before proceeding.
- If you receive a `401 Unauthorized` response, request a new token and retry the request once. Do not retry on consecutive 401 responses.
