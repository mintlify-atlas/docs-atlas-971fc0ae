[Terraform](https://www.terraform.io/), developed by HashiCorp, is an infrastructure as code tool that lets you define and provision infrastructure using a declarative configuration language. Airbyte's Terraform provider is built on top of the [Airbyte API](https://reference.airbyte.com) and lets you version-control your sources, destinations, and connections alongside the rest of your infrastructure.

## Quick example

The following example shows a minimal Terraform configuration that creates a Stripe source, a BigQuery destination, and connects them.

<Steps>
  <Step title="Configure the provider">
    ```hcl
    terraform {
      required_providers {
        airbyte = {
          source  = "airbytehq/airbyte"
          version = "~> 1.0"
        }
      }
    }

    provider "airbyte" {
      client_id     = var.airbyte_client_id
      client_secret = var.airbyte_client_secret
    }
    ```
  </Step>

  <Step title="Create a source">
    ```hcl
    resource "airbyte_source" "stripe" {
      name         = "Stripe Production"
      workspace_id = var.workspace_id
      definition_id = "e094cb9a-26de-4645-8761-65c0c425d1de"

      configuration = jsonencode({
        account_id   = var.stripe_account_id
        client_secret = var.stripe_secret_key
        start_date   = "2023-01-01T00:00:00Z"
      })
    }
    ```
  </Step>

  <Step title="Create a destination">
    ```hcl
    resource "airbyte_destination" "bigquery" {
      name         = "BigQuery Data Warehouse"
      workspace_id = var.workspace_id
      definition_id = "22f6c74f-5699-40ff-833c-4a879ea40133"

      configuration = jsonencode({
        project_id       = var.gcp_project_id
        dataset_id       = "airbyte_data"
        dataset_location = "US"
      })
    }
    ```
  </Step>

  <Step title="Create a connection">
    ```hcl
    resource "airbyte_connection" "stripe_to_bigquery" {
      name           = "Stripe to BigQuery"
      source_id      = airbyte_source.stripe.source_id
      destination_id = airbyte_destination.bigquery.destination_id

      schedule = {
        schedule_type   = "cron"
        cron_expression = "0 0 * * *"
      }

      namespace_definition = "destination"
      prefix               = "stripe_"
    }
    ```
  </Step>

  <Step title="Apply the configuration">
    ```bash
    terraform init
    terraform plan
    terraform apply
    ```
  </Step>
</Steps>

## Authentication

The Terraform provider authenticates using the same client credentials as the Airbyte API. Set up your credentials following the [API access configuration](/developers/api-documentation) guide, then pass them to the provider using variables or environment variables.

```hcl
# Using environment variables (recommended for CI/CD)
# export AIRBYTE_CLIENT_ID=your_client_id
# export AIRBYTE_CLIENT_SECRET=your_client_secret

provider "airbyte" {}
```

```bash
export AIRBYTE_CLIENT_ID="your_client_id"
export AIRBYTE_CLIENT_SECRET="your_client_secret"
```

<Tip>
For CI/CD pipelines, store credentials in your secrets manager and inject them as environment variables rather than hardcoding them in Terraform files.
</Tip>

## Migrating to provider 1.0

Version 1.0 of the provider replaced connector-specific resource types (like `airbyte_source_stripe`) with two generic resource types: `airbyte_source` and `airbyte_destination`. If you are on a pre-1.0 version, see the [migration guide](https://registry.terraform.io/providers/airbytehq/airbyte/latest/docs/guides/v1_migration_guide) and the [pre-1.0 reference](https://registry.terraform.io/providers/airbytehq/airbyte/latest/docs).
