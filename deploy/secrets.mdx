---
title: 'Managing secrets'
description: 'Create and manage Kubernetes secrets for Airbyte deployments, including database credentials, storage keys, and connector secret managers.'
---

Airbyte's Helm chart requires sensitive values — API keys, database credentials, storage access keys — to be stored as Kubernetes secrets before installation begins. Each integration you configure will specify which secrets it expects.

<Tip>
While you can name your secret anything, Airbyte recommends using `airbyte-config-secrets` as the secret name. This name is referenced in the default values.yaml configuration, which reduces the number of overrides you need to make.
</Tip>

## Create a Kubernetes secret

You can create secrets from a YAML manifest or directly using `kubectl`.

<Tabs>
  <Tab title="YAML manifest">
    Create a file named `secrets.yaml`:

    ```yaml
    apiVersion: v1
    kind: Secret
    metadata:
      name: airbyte-config-secrets
    type: Opaque
    stringData:
      # Examples — replace with your actual values
      key-1: "value-1"
      key-2: "value-2"
    ```

    Apply it to your cluster:

    ```bash
    kubectl apply -f secrets.yaml -n airbyte
    ```
  </Tab>
  <Tab title="kubectl">
    ```bash
    kubectl create secret generic airbyte-config-secrets \
      --from-literal=key-1='value-1' \
      --from-literal=key-2='value-2' \
      --namespace airbyte
    ```
  </Tab>
</Tabs>

## Secret keys by integration

The following sections show the secret keys required for each integration. Include only the keys relevant to your deployment.

### Database credentials

```yaml
stringData:
  database-host: database.internal
  database-port: "5432"
  database-name: airbyte
  database-user: airbyte
  database-password: your-password
```

### Instance admin credentials

```yaml
stringData:
  instance-admin-email: admin@company.example
  instance-admin-password: your-password
```

### Enterprise license key

```yaml
stringData:
  license-key: xxxxx.yyyyy.zzzzz
```

### AWS S3 storage

```yaml
stringData:
  s3-access-key-id: AKIAIOSFODNN7EXAMPLE
  s3-secret-access-key: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
```

### AWS Secrets Manager

```yaml
stringData:
  aws-secret-manager-access-key-id: AKIAIOSFODNN7EXAMPLE
  aws-secret-manager-secret-access-key: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
```

### GCP credentials (for GCS and Google Secret Manager)

For GCP, you typically store a service account JSON file as a secret key:

```yaml
stringData:
  gcp.json: |
    {
      "type": "service_account",
      "project_id": "my-project",
      ...
    }
```

Or create the secret directly from a JSON file:

```bash
kubectl create secret generic airbyte-config-secrets \
  --from-file=gcp.json \
  --namespace airbyte
```

### Azure Blob Storage

```yaml
stringData:
  azure-blob-store-connection-string: "DefaultEndpointsProtocol=https;AccountName=myaccount;AccountKey=mykey==;EndpointSuffix=core.windows.net"
```

### Azure Key Vault

```yaml
stringData:
  azure-key-vault-client-id: 3fc863e9-4740-4871-bdd4-456903a04d4e
  azure-key-vault-client-secret: KWP6egqixiQeQoKqFZuZq2weRbYoVxMH
```

### SSO OIDC credentials

```yaml
stringData:
  client-id: e83bbc57-1991-417f-8203-3affb47636cf
  client-secret: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
```

## Reference secrets from values.yaml

Once your Kubernetes secret exists, reference it in your `values.yaml` by providing the secret name and the key within that secret. For example:

```yaml
global:
  database:
    secretName: airbyte-config-secrets
    passwordSecretKey: database-password

  storage:
    storageSecretName: airbyte-config-secrets

  auth:
    instanceAdmin:
      secretName: airbyte-config-secrets
      emailSecretKey: instance-admin-email
      passwordSecretKey: instance-admin-password

  enterprise:
    secretName: airbyte-config-secrets
    licenseKeySecretKey: license-key
```

## Update an existing secret

To update a value in an existing secret, delete and recreate it, or use `kubectl patch`:

```bash
kubectl patch secret airbyte-config-secrets \
  -n airbyte \
  --type='json' \
  -p='[{"op": "replace", "path": "/data/database-password", "value": "'$(echo -n new-password | base64)'"}]'
```

After updating secrets, run `helm upgrade` to apply the changes to your Airbyte deployment.
