---
title: "Connector Metadata File"
description: "Reference for the metadata.yaml file that every Airbyte connector requires."
---

Every connector in the Airbyte repository has a `metadata.yaml` file in its root directory. This file tells Airbyte's tooling and registry services how to handle the connector: which Docker image to use, which registries to publish it to, and what test credentials are needed in CI.

## Example

```yaml
data:
  allowedHosts:
    hosts:
      - ${host}
      - ${tunnel_method.tunnel_host}
  connectorSubtype: database
  connectorType: source
  definitionId: decd338e-5647-4c0b-adf4-da0e75f5a750
  dockerImageTag: 2.0.28
  maxSecondsBetweenMessages: 7200
  dockerRepository: airbyte/source-postgres
  githubIssueLabel: source-postgres
  icon: postgresql.svg
  license: ELv2
  name: Postgres
  tags:
    - language:java
    - language:python
  registryOverrides:
    cloud:
      dockerRepository: airbyte/source-postgres-strict-encrypt
      enabled: true
    oss:
      enabled: true
  supportLevel: certified
  documentationUrl: https://docs.airbyte.com/integrations/sources/postgres
metadataSpecVersion: "1.0"
```

## Top-level fields

| Field | Description |
|---|---|
| `connectorType` | `source` or `destination` |
| `connectorSubtype` | Category: `api`, `database`, `file`, `custom` |
| `definitionId` | Stable UUID that uniquely identifies this connector |
| `dockerImageTag` | Current Docker image version tag |
| `dockerRepository` | Docker Hub repository path |
| `name` | Human-readable connector name |
| `icon` | SVG icon filename |
| `license` | Connector license (e.g. `ELv2`, `MIT`) |
| `documentationUrl` | Link to the connector's documentation page |
| `githubIssueLabel` | GitHub label used for connector issues |
| `supportLevel` | `certified`, `community`, or `archived` |
| `maxSecondsBetweenMessages` | Timeout before assuming the connector is stalled |
| `allowedHosts` | Hosts the connector is permitted to connect to |

## `registryOverrides`

Controls which Airbyte registries include this connector.

```yaml
registryOverrides:
  cloud:
    dockerRepository: airbyte/source-postgres-strict-encrypt
    enabled: true
  oss:
    enabled: true
```

- `enabled: true` under `oss` includes the connector in `oss_registry.json`
- `enabled: true` under `cloud` includes it in `cloud_registry.json`
- `dockerRepository` under `cloud` lets you use a different (stricter) image for Airbyte Cloud

When you update `metadata.yaml`, the metadata service automatically regenerates both registry files.

## `tags`

Tags provide additional context and improve discoverability.

```yaml
tags:
  - language:java
  - language:python
  - keyword:database
  - keyword:SQL
```

- `language:*` tags indicate programming languages used in the connector (auto-generated by tooling)
- `keyword:*` tags make the connector searchable by topic

## `releases`

Documents breaking changes so Airbyte can warn users before they upgrade.

### `breakingChanges`

```yaml
releases:
  breakingChanges:
    1.0.0:
      message: "This version removes ApiKey authentication, which is now deprecated.
        Users currently using ApiKey auth will need to reauthenticate with OAuth
        after upgrading to continue syncing."
      upgradeDeadline: "2023-12-31"
```

Each entry must include:

| Field | Description |
|---|---|
| `message` | What changed, who is affected, why it's better, what the user must do |
| `upgradeDeadline` | Date by which users should upgrade (`YYYY-MM-DD`). Typically 2 weeks for simple changes |

<Warning>
Do not tie `upgradeDeadline` to an upstream API deprecation date. Users pinned to older connector versions won't receive future fixes. Set the deadline based on how long the migration takes, not how long the old API version works.
</Warning>

### `scopedImpact`

When only certain streams are affected by a breaking change, use `scopedImpact` to limit the scope:

```yaml
releases:
  breakingChanges:
    1.0.0:
      message: "This version changes the cursor for the `users` stream. After upgrading, please reset the stream."
      upgradeDeadline: "2023-12-31"
      scopedImpact:
        - scopeType: stream
          impactedScopes: ["users"]
```

Users who aren't syncing the `users` stream can safely ignore this breaking change.

| `scopeType` | Value type | Description |
|---|---|---|
| `stream` | `list[str]` | Names of affected streams |

## `connectorTestSuitesOptions`

Declares which test suites CI should run for this connector.

```yaml
connectorTestSuitesOptions:
  - suite: unitTests
  - suite: acceptanceTests
    testSecrets:
      - name: SECRET_SOURCE-MYCONNECTOR_CREDS
        fileName: config.json
        secretStore:
          type: GSM
          alias: airbyte-connector-testing-secret-store
```

Supported suites:

| Suite | Description |
|---|---|
| `unitTests` | Unit tests in `unit_tests/` (Python) or via Gradle `test` task (Java) |
| `integrationTests` | Integration tests in `integration_tests/` or via Gradle `integrationTest` task |
| `acceptanceTests` | Connector Acceptance Tests (CAT), configured in `acceptance-test-config.yml` |

### `testSecrets`

Each test secret has:

| Field | Required | Description |
|---|---|---|
| `name` | Yes | Secret name in Google Secret Manager |
| `fileName` | No | File path relative to `secrets/` where the secret is saved locally (e.g. `config.json`) |
| `secretStore.type` | Yes | Secret store type (currently only `GSM`) |
| `secretStore.alias` | Yes | GSM alias (use `airbyte-connector-testing-secret-store`) |

<Note>
Adding a new secret to the secret store requires manual intervention from an Airbyte team member. Reach out to your PR reviewer.
</Note>

## `remoteRegistries`

Optional configuration for publishing the connector to external package registries.

```yaml
remoteRegistries:
  pypi:
    enabled: true
    packageName: airbyte-source-myconnector
```

<Note>
No automated publishing occurs currently. This configuration is reserved for future use.
</Note>
