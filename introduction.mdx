## What is Airbyte?

Airbyte is a data integration platform that extracts data from sources (databases, APIs, files) and loads it into destinations (data warehouses, lakes, databases). It follows the ELT (Extract, Load, Transform) pattern — moving raw data first, then transforming it in the destination.

**Two ways to use Airbyte:**

- **Airbyte Cloud** — Fully managed SaaS, no infrastructure to maintain. [Sign up free](https://cloud.airbyte.com/signup).
- **Airbyte Self-Managed** — Deploy on your own Kubernetes cluster with full control over your data and infrastructure.

## Architecture at a Glance

Airbyte is built as a set of microservices deployed on Kubernetes:

- **Config API Server** — Main controller, REST API, and web UI host
- **Temporal** — Workflow scheduler and orchestrator
- **Workers** — Execute connection sync and check jobs
- **Workload API** — Manages discrete connector operation pods
- **Connectors** — Docker containers implementing the Airbyte Protocol

Each connector is a Docker image that implements four operations: `spec`, `check`, `discover`, and `read` (for sources) or `write` (for destinations).
