Airbyte's Google Analytics 4 (GA4) source connector uses the [Google Analytics Data API](https://developers.google.com/analytics/devguides/reporting/data/v1) to replicate event data, session reports, demographic reports, ecommerce metrics, and custom reports. It works with both GA4 and Google Analytics 360 (GA360) properties.

<Note>
This connector only supports Google Analytics 4 properties. Universal Analytics (UA) properties with Tracking IDs like `UA-123...-1` are not supported.
</Note>

## Prerequisites

- A Google Analytics account with access to the GA4 property you want to sync
- The numeric **Property ID** of your GA4 property
- For Airbyte Open Source: a Google Cloud service account with Viewer access to the GA4 property and the Analytics Data API enabled

## Setup guide

<Tabs>
  <Tab title="Airbyte Cloud (OAuth)">
    <Steps>
      <Step title="Set up the connector">
        1. In Airbyte, go to **Sources** > **+ New source** and select **Google Analytics 4 (GA4)**.
        2. Click **Authenticate via Google (OAuth)** and sign in to your Google account.
        3. Enter your numeric **Property ID** (find this in GA4 under **Admin** > **Property Settings**).
        4. (Optional) Set a **Start Date** in `YYYY-MM-DD` format.
        5. (Optional) Configure **Custom Reports** (see [Custom reports](#custom-reports)).
        6. (Optional) Set a **Data Request Interval** (1–364 days). Smaller values reduce sampling but slower syncs. Default: 1 day.
        7. Click **Set up source**.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Airbyte Open Source (Service Account)">
    <Steps>
      <Step title="Create a service account">
        1. Go to the [Service Accounts](https://console.developers.google.com/iam-admin/serviceaccounts) page in Google Cloud Console.
        2. Create a new service account with the **Viewer** role.
        3. Create and download a **JSON key** for the service account.
      </Step>
      <Step title="Enable APIs">
        Enable the following APIs in your Google Cloud project:
        - [Google Analytics Reporting API](https://console.developers.google.com/apis/api/analyticsreporting.googleapis.com/overview)
        - [Google Analytics API](https://console.developers.google.com/apis/api/analytics.googleapis.com/overview)
        - [Google Analytics Data API](https://console.developers.google.com/apis/api/analyticsdata.googleapis.com/overview)
      </Step>
      <Step title="Grant property access">
        In Google Analytics, go to **Admin** > **Property access management** > **+** > **Add users**. Add the service account email (e.g., `...@...iam.gserviceaccount.com`) with at least the **Viewer** role.
      </Step>
      <Step title="Set up the connector">
        1. In Airbyte, go to **Sources** > **+ New source** and select **Google Analytics 4 (GA4)**.
        2. Select **Service Account Key Authentication** and paste your JSON key.
        3. Enter your numeric **Property ID**.
        4. (Optional) Configure start date, custom reports, data request interval, and lookback window.
        5. Click **Set up source**.
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Custom reports

Custom reports let you query specific combinations of dimensions and metrics beyond the preconfigured streams.

Define custom reports as a JSON array in the **Custom Reports** field:

```json
[
  {
    "name": "User Engagement Report",
    "dimensions": ["city"],
    "metrics": ["sessions", "bounceRate"]
  }
]
```

You can also include optional `cohortSpec` and `pivots` fields. See the [GA4 Dimensions & Metrics Explorer](https://ga-dev-tools.google/ga4/dimensions-metrics-explorer/) for a full list of available dimensions and metrics.

## Data sampling

GA4 may apply sampling when the volume of data processed for a query exceeds Google's compute thresholds. To minimize sampling:

- Set the **Data Request Interval** to a small value (1 day is ideal).
- Avoid using dimensions like `month` or `yearMonth` with a 1-day interval — this can produce duplicate rows. In that case, increase the interval to 364 days.

## Configuration reference

<ParamField path="property_ids" type="array" required>
  List of numeric GA4 Property IDs to sync (e.g., `["123456789"]`).
</ParamField>

<ParamField path="credentials" type="object" required>
  Authentication credentials. Either OAuth tokens or a Service Account JSON key.
</ParamField>

<ParamField path="start_date" type="string">
  Start date for data replication in `YYYY-MM-DD` format. Defaults to 2 years ago.
</ParamField>

<ParamField path="custom_reports" type="array">
  Array of custom report definitions with `name`, `dimensions`, and `metrics` fields.
</ParamField>

<ParamField path="window_in_days" type="integer">
  The time window in days for each API request. Range: 1–364. Default: 1.
</ParamField>

<ParamField path="lookback_window" type="integer">
  Number of days before the last cursor to re-fetch on each sync. Useful for attribution changes.
</ParamField>

<ParamField path="keep_empty_rows" type="boolean">
  Whether to return rows where all metrics equal 0. Default: false.
</ParamField>
