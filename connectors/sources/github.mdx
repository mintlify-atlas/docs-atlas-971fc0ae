title: GitHub Source
description: Replicate issues, pull requests, commits, workflows, and more from GitHub repositories using OAuth or a Personal Access Token.
---

Airbyte's GitHub source connector replicates repository data from GitHub. It supports both full refresh and incremental sync across dozens of streams covering issues, pull requests, commits, deployments, workflows, and more.

## Prerequisites

- One or more GitHub repositories (public or private)
- For Airbyte Cloud: OAuth or a Personal Access Token
- For Airbyte Open Source: a Personal Access Token

## Setup guide

<Steps>
  <Step title="Create a GitHub Personal Access Token (Open Source / PAT auth)">
    1. Log in to [GitHub](https://github.com) and go to [Settings > Developer settings > Personal access tokens](https://github.com/settings/tokens).
    2. Click **Generate new token** and select the **repo** scope at minimum.
    3. Add additional scopes as needed for the streams you want (see [Permissions and scopes](#permissions-and-scopes)).
    4. Copy the generated token.

    To load-balance across multiple tokens (useful for large repos hitting rate limits), enter multiple tokens separated by commas.
  </Step>
  <Step title="Set up the GitHub connector in Airbyte">
    1. In Airbyte, go to **Sources** > **+ New source** and select **GitHub**.
    2. Enter a name for the connector.
    3. Authenticate:
       - **Airbyte Cloud**: Click **Authenticate your GitHub account** for OAuth, or select **Personal Access Token** and enter your PAT.
       - **Airbyte Open Source**: Select **Personal Access Token** and enter your PAT.
    4. For **GitHub Repositories**, enter repositories in the format `owner/repo`. Separate multiple repos with spaces. To sync all repos in an org, use `org/*`.
    5. (Optional) Set a **Start date** for streams that support it.
    6. (Optional) Set **Branch** to limit commit syncing to specific branches (format: `owner/repo/branch`).
    7. Click **Set up source**.
  </Step>
</Steps>

## Supported sync modes

| Mode | Supported |
|------|-----------|
| Full Refresh - Overwrite | Yes |
| Full Refresh - Append | Yes |
| Incremental - Append | Yes |
| Incremental - Append + Deduped | Yes |

## Supported streams

### Full refresh streams

These streams return all data on every sync:

- `assignees`, `branches`, `collaborators`
- `contributor_activity`
- `issue_labels`, `issue_timeline_events`
- `organizations`
- `pull_request_commits`
- `repositories`, `tags`, `teams`, `team_members`, `team_memberships`, `users`

### Incremental streams

These streams only emit new records based on the **Start Date**:

- `comments`, `commit_comments`, `commit_comment_reactions`
- `commits`
- `deployments`, `events`
- `issue_comment_reactions`, `issue_events`, `issue_milestones`, `issue_reactions`, `issues`
- `project_cards`, `project_columns`, `projects`, `projects_v2`
- `pull_request_comment_reactions`, `pull_request_stats`, `pull_requests`
- `releases`
- `repositories`
- `review_comments`, `reviews`
- `stargazers`
- `workflow_jobs`, `workflow_runs`, `workflows`

<Note>
The `workflow_runs` and `workflow_jobs` streams also re-read records from the past 30 days to capture re-run workflows.
</Note>

## Streams that ignore Start Date

The following streams sync all records regardless of the configured **Start Date** because GitHub's API does not return date fields for them:

`assignees`, `branches`, `collaborators`, `issue_labels`, `organizations`, `pull_request_commits`, `repositories`, `tags`, `teams`, `users`

## Permissions and scopes

For a Personal Access Token, ensure the following scopes are granted based on which streams you need:

- **`repo`**: Required for all private repository streams
- **`read:org`**: Required for `teams`, `team_members`, `team_memberships`
- **`read:user`**: Required for user-related streams
- **`workflow`**: Required for `workflow_runs`, `workflow_jobs`, `workflows`
- **`read:project`**: Required for `projects_v2`

For OAuth, Airbyte requests: `repo`, `read:org`, `read:repo_hook`, `read:user`, `read:discussion`, `read:project`, `workflow`.

## Rate limits

GitHub allows 5,000 requests per hour per token (15,000 for GitHub Enterprise Cloud organizations). REST API and GraphQL API rate limits are counted separately.

Streams using the GraphQL API include `pull_request_stats`, `reviews`, `pull_request_comment_reactions`, `issue_reactions`, `releases`, and `projects_v2`.

<Tip>
To work around rate limits: use incremental sync, set a higher sync interval, or split large repositories across separate connections. You can also provide multiple Personal Access Tokens separated by commas.
</Tip>

## Configuration reference

<ParamField path="credentials" type="object" required>
  Authentication credentials. Either OAuth tokens or a Personal Access Token.
</ParamField>

<ParamField path="repositories" type="array" required>
  List of GitHub repositories in `owner/repo` format. Use `owner/*` to sync an entire organization.
</ParamField>

<ParamField path="start_date" type="string">
  The start date for incremental streams, in ISO 8601 format (`YYYY-MM-DDTHH:MM:SSZ`).
</ParamField>

<ParamField path="branch" type="array">
  List of branches to include for the `commits` stream, in `owner/repo/branch` format.
</ParamField>
